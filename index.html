7Human Layer — طبقة الإنسان (الأكثر خارجية)▼
🇬🇧 النص الأصلي
Being aware of human threats like spamming, phishing, and any other clever form of social engineering.
🇸🇦 الترجمة
الوعي بالتهديدات البشرية:
• Spamming = البريد العشوائي غير المرغوب فيه
• Phishing = التصيد الاحتيالي (خداع المستخدم لسرقة بياناته)
• Social Engineering = الهندسة الاجتماعية (التلاعب النفسي بالإنسان)
💡 ملاحظة
الإنسان = أضعف حلقة في الأمن السيبراني. هذه الطبقة الأكثر خارجية (الأولى في مواجهة الهجمات).
6Perimeter Security — أمن المحيط▼
🇬🇧 النص الأصلي
Prevents suspicious activity from entering the network by protecting the gateway with firewalls, data encryption, anti-virus software.
🇸🇦 الترجمة
تمنع النشاط المشبوه من الدخول عبر:
• Firewalls = جدران الحماية
• Data Encryption = تشفير البيانات
• Anti-Virus Software = برامج مكافحة الفيروسات
5Network Security — أمن الشبكة▼
🇬🇧 النص الأصلي
We need to provide people with the minimum amount of "user privilege" possible for them to do their job and nothing more.
🇸🇦 الترجمة
منح الحد الأدنى من الصلاحيات اللازمة فقط.
💡 المبدأ
Least Privilege = مبدأ الحد الأدنى من الصلاحيات — أعطِ كل مستخدم فقط ما يحتاجه لعمله.
4Endpoint Security — أمن نقاط النهاية▼
🇸🇦 الترجمة
تحمي الاتصال بين الشبكة وأجهزتها عبر:
• Antivirus Software = برامج مكافحة الفيروسات
• Web Content Filtering = تصفية محتوى الويب
• Application Controls = التحكم في التطبيقات
3Application Security — أمن التطبيقات▼
🇸🇦 الترجمة
اختبار التطبيقات وإضافة ميزات لمنع Vulnerabilities (الثغرات) وإغلاقها بـPatch (تصحيح). التحديث الدوري لعدم ترك أي نظام عرضة لـOld Exploit (ثغرة قديمة).
2Data Security — أمن البيانات▼
🇸🇦 الترجمة
• Backups = نسخ احتياطية موثوقة
• Encryption = تشفير البيانات
• Two-Factor Authentication (2FA) = المصادقة الثنائية
1Mission Critical Assets — الأصول الحيوية (الأكثر داخلية)▼
🇸🇦 الترجمة
أي Asset (أصل/مورد) لا تستطيع المنظمة العمل بدونه: أجهزة (Computers) + برامج (Software) + بيانات (Data). الطبقة الأعمق = الأكثر حمايةً.
🌱 مصدر الثغرات — Origin of Vulnerabilities
3Design / Implementation / Configuration — المصادر الثلاثة▼
1. Design Vulnerability
ثغرة التصميمعيوب في التصميم الأصلي للنظام.
مثال: عدم استخدام HTTPS من البداية.
2. Implementation Vulnerability
ثغرة التنفيذ / التطبيقأخطاء أثناء تنفيذ البرنامج.
مثال: تثبيت البرنامج بطريقة خاطئة.
3. Configuration Vulnerability
ثغرة الإعداد / الضبطإعداد خاطئ أو استخدام الإعدادات الافتراضية.
مثال: عدم تغيير Default Password (كلمة المرور الافتراضية).

🔍 Penetration Testing — اختبار الاختراق
8الخطوات الثماني كاملة مع الترجمة والأدوات▼

Planning & Preparation — التخطيط والتحضير

تحديد Scope (النطاق) + الأهداف + الإطار الزمني. الحصول على Legal Permissions (الأذونات القانونية) وتوقيع العقد مع Client (العميل).
أدوات: توقيع العقود — لا أدوات تقنية.

Reconnaissance — الاستطلاع

جمع سلبي للمعلومات المتاحة للعموم عن الهدف (Target): أسماء نطاقات (Domain Names)، عناوين IP، تقنيات، معلومات موظفين.
يُسمى OSINT = Open Source Intelligence = استخبارات المصادر المفتوحة
أدوات: Google Dorking, Maltego, Recon-ng, TheHarvester, Metasploit

Scanning & Enumeration — الفحص والتعداد

Network Scanning = فحص الشبكة لاكتشاف Active Hosts (المضيفين النشطين).
Enumeration = تعداد Open Ports (المنافذ المفتوحة) والـServices (الخدمات).
أدوات: Nmap, PowerView

Vulnerability Assessment — تقييم الثغرات

استخدام Automated Scanners (ماسحات آلية) لتحديد الثغرات المعروفة.
Noisy (مزعج) = يُولِّد كميات كبيرة من حركة البيانات ويُكتشف بسهولة.
أدوات: Nessus, Metasploit, Greenbone

Exploitation — الاستغلال

بعد الوصول الأولي (Initial Access)، استغلال التطبيقات، الشبكة، أنظمة التشغيل، البشر، Misconfigurations (الإعدادات الخاطئة).
أدوات: sqlmap, John the Ripper, Burp Suite

Post-Exploitation — ما بعد الاستغلال

• Privilege Escalation = رفع الصلاحيات
• Persistence Mechanisms = آليات الاستمرارية للبقاء في النظام
• Leave no evidence = لا تترك أدلة — أزل الملفات والإعدادات.
أدوات: BloodHound, Responder, Seatbelt, Mimikatz

Reporting — التقرير

إنشاء تقرير يُفصِّل الثغرات المكتشفة وكيفية Remediation (معالجتها).
أدوات: Dradis, Faraday

Remediation & Follow-Up — المعالجة والمتابعة

مناقشة نتائج التقييم (Assessment Outcome) مع العميل. قد يشمل Re-testing = إعادة الاختبار.

⚡ NIST — 4 خطوات ملخصة
1. Plan (خطِّط) — تحديد النطاق والأهداف
2. Discover (اكتشف) — جمع المعلومات والاستطلاع
3. Attack (هاجم) — استغلال الثغرات
4. Report (أبلِّغ) — توثيق النتائج وتسليمها للعميل
✏️ تطبيقات
OSINT تُستخدم في مرحلة: MCQ
✅ B — Reconnaissance = الاستطلاع
في Post-Exploitation يجب إبقاء الأدلة. T/F
❌ خطأ"Leave no evidence" — أزل جميع الأدلة.
Nessus تُستخدم في مرحلة: MCQ
✅ C — Vulnerability Assessment
📊 15 خاصية للثغرات — Key Characteristics
5الخصائص 1–5▼
1. Exploitability
قابلية الاستغلالThe ease with which a vulnerability can be exploited.
= مدى سهولة استغلال الثغرة.
أداة استغلال جاهزة للعموم = Exploitability عالية.
2. Impact
الأثر / التأثيرPotential consequences if exploited.
= النتائج المحتملة إذا استُغلت الثغرة.
CIA Triad: Confidentiality (سرية) + Integrity (سلامة) + Availability (توافر).
3. Scope
النطاقThe extent of systems affected.
= نطاق الأنظمة المتأثرة.
ثغرة في مكتبة شائعة = Scope (نطاق) واسع.
4. Attack Vector
ناقل الهجوم (مسار الهجوم)The path by which an attacker exploits the vulnerability.
= المسار الذي يستخدمه المهاجم.
Network (شبكة), Physical (مادي), Social Engineering (هندسة اجتماعية).
5. Complexity
التعقيدThe level of difficulty to exploit.
= مستوى صعوبة استغلال الثغرة.
ثغرة تتطلب معرفة متقدمة = Complexity عالٍ.
5الخصائص 6–10▼
6. Privileges Required
الصلاحيات / الامتيازات المطلوبةLevels: None (لا شيء) / Low-user (منخفضة) / High-admin (عالية-مدير).
⚠️ None = الأخطر! أي شخص يستطيع الاستغلال.
7. Detection Difficulty
صعوبة الاكتشافHow easily the vulnerability can be detected.
= مدى صعوبة اكتشاف الثغرة.
ثغرة في ميزة نادرة الاستخدام = أصعب اكتشافاً.
8. Remediation Difficulty
صعوبة المعالجة / الإصلاحThe effort required to fix the vulnerability.
= الجهد اللازم لإصلاح الثغرة.
ثغرة تتطلب إعادة تصميم كامل = صعبة المعالجة.
9. Dependency
الاعتماديةRelationship between the vulnerability and other components.
= علاقة الثغرة بمكونات أخرى.
ثغرة في Third-party Library (مكتبة خارجية).
10. Time Sensitivity
الحساسية الزمنية (الإلحاح)The urgency to address the vulnerability.
= مدى إلحاحية التعامل مع الثغرة.
Zero-Day (ثغرة يوم الصفر) مع استغلال نشط = اهتمام فوري!
5الخصائص 11–15▼
11. Persistence
الاستمرارية (البقاء في النظام)Whether the vulnerability allows attacker to maintain access.
= هل تسمح للمهاجم بالبقاء؟
ثغرة تسمح بتثبيت Malware (برمجية خبيثة) = Persistence.
12. Environmental Factors
العوامل البيئية (السياق المحيط)The context in which the vulnerability exists.
= البيئة والإعداد المحيط بالثغرة.
نظام مكشوف على الإنترنت أخطر من نظام Isolated (معزول).
13. Public Awareness
الوعي العام (مدى المعرفة بالثغرة)The level of knowledge the vulnerability has received.
= مستوى الانتباه الذي نالته الثغرة.
ثغرة أُعلنت في مؤتمر أمني كبير = Public Awareness عالٍ.
14. Patch Availability
توفر التصحيح / الإصلاحWhether a fix or patch is available.
= هل يتوفر Patch (تصحيح) للثغرة؟
ثغرة بلا Patch = أخطر من ثغرة لها Patch.
15. Attack Surface
سطح الهجوم (مساحة التعرض)The exposure of the vulnerability to potential attackers.
= مدى تعرض الثغرة للمهاجمين.
تطبيق ويب مكشوف = Attack Surface أكبر من نظام داخلي.
✏️ تطبيقات
Zero-Day مع active exploitation يرتبط بخاصية: MCQ
✅ B — Time Sensitivity
Privileges Required = None (لا صلاحيات) هي الأقل خطورة. T/F
❌ خطأNone = الأخطر! لأن أي شخص يستطيع الاستغلال بدون صلاحيات.
!CVSS Risk Score + Risk Matrix — درجة المخاطرة▼
🇸🇦 شرح
CVSS = Common Vulnerability Scoring System = نظام تسجيل درجة الثغرة
يستخدم الخصائص السابقة لتحديد Risk Score (درجة المخاطرة) لكل ثغرة.
💡 Risk Matrix (Impact × Likelihood)
🔴 Critical — Unacceptable (9) = خطر حرج غير مقبول — يُعالج فوراً
🟠 High Risk (7-8) = خطر عالٍ — يُعالج بسرعة (Promptly)
🟡 Medium Risk (4-6) = خطر متوسط — يُعالج حسب الوقت
🟢 Low Risk — Acceptable (1-3) = خطر منخفض — عند توفر الموارد
🗂️ تصنيف الثغرات — 5 طرق
1By Domain — حسب النطاق▼
Software Vulnerabilities
ثغرات البرمجيات• Buffer Overflows = فيض المخزن المؤقت
• SQL Injection = حقن SQL
• XSS = Cross-site Scripting = برمجة عبر المواقع
• Insecure Deserialization = إلغاء تسلسل غير آمن
Hardware Vulnerabilities
ثغرات الأجهزة المادية• Firmware Vulnerabilities = ثغرات البرامج الثابتة المدمجة في الجهاز
• Side-channel Attacks = هجمات القناة الجانبية
Network Vulnerabilities
ثغرات الشبكة• Weak Encryption = تشفير ضعيف
• Misconfigured Firewalls = جدران حماية مُعدَّة خطأ
• Insecure Network Protocols = بروتوكولات شبكة غير آمنة
Human / Organizational Vulnerabilities
ثغرات بشرية / تنظيمية• Social Engineering = الهندسة الاجتماعية
• Lack of Security Awareness = ضعف الوعي الأمني
• Poor Password Management = إدارة كلمات مرور سيئة
2By Type of Flaw — حسب نوع العيب▼
Input Validation Vulnerabilitiesثغرات التحقق من المدخلات
الفشل في التحقق من User Input (مدخلات المستخدم) → SQL Injection + XSS.
Authentication & Authorization Vulnerabilitiesثغرات المصادقة والتفويض
• Authentication = المصادقة: كلمات مرور ضعيفة
• Authorization = التفويض: Missing Access Controls (غياب ضوابط الوصول)
Memory Management Vulnerabilitiesثغرات إدارة الذاكرة
• Buffer Overflows = فيض المخزن المؤقت
• Use-after-free errors = أخطاء استخدام الذاكرة بعد تحريرها
Configuration Vulnerabilitiesثغرات الإعداد
• Default Passwords = كلمات مرور افتراضية
• Open Ports = منافذ مفتوحة غير ضرورية
3By Severity — حسب الخطورة▼
🔴 Critical — حرج
خطر بالغ. تُعالج فوراً (Immediately).
🟠 High — عالٍ
أضرار كبيرة. تُعالج بسرعة (Promptly).
🟡 Medium — متوسط
خطر معتدل. تُعالج حسب الوقت (As time permits).
🟢 Low — منخفض
خطر ضئيل. تُعالج عند توفر الموارد (When resources available).

🛡️ استراتيجيات التخفيف — المحاضرة 6
!⚠️ Mitigation مقابل Remediation — الفرق الأهم▼
Mitigation — التخفيف
تقليل المخاطر مؤقتاً حتى يُوجد حل دائم.
= Temporary (مؤقت)
Remediation — المعالجة
تصحيح الثغرة فعلياً بالوصول إلى Root Cause (السبب الجذري).
= Permanent (دائم) — الهدف النهائي دائماً!
💡 متى Mitigation بدلاً من Remediation؟
• لا يتوفر Patch (تصحيح) فوري
• الثغرة ليست خطيرة كفاية
• لا توجد موارد أو كوادر كافية (No Bandwidth)
• Compatibility Issues (مشاكل توافق تقني) تمنع الإصلاح
1Patch Management — إدارة التصحيحات▼
🇸🇦 التعريف
عملية الحصول على (Acquiring) واختبار (Testing) وتثبيت (Installing) تحديثات البرامج (Patches = تصحيحات) لإصلاح الثغرات وتحسين استقرار النظام (Stability).
💡 Best Practices — أفضل الممارسات
• إنشاء Policy (سياسة) لإدارة التصحيحات
• الاحتفاظ بـInventory (جرد) لجميع البرامج
• تحديد الأولويات (Prioritize) بناءً على المخاطر
• اختبار في بيئة Non-production (غير إنتاجية)
• أتمتة (Automate) العملية
• وجود Rollback Plan (خطة تراجع)
أدوات: WSUS, SCCM
2Security Configuration — الإعداد الأمني▼
🇸🇦 التعريف
Hardening = تقوية الأنظمة بتطبيق إعدادات آمنة لتقليل Attack Surface (سطح الهجوم).
💡 أفضل الممارسات
• اتباع CIS Benchmarks (معايير أمنية مرجعية)
• تعطيل (Disable) الخدمات والـPorts (منافذ) غير الضرورية
• تطبيق Least Privilege (الحد الأدنى من الصلاحيات)
• فرض Strong Password Policies (سياسات كلمات مرور قوية)
• تطبيق Logging and Auditing (التسجيل والمراجعة)
أدوات: Group Policy, أدوات إدارة الإعداد
3Access Control — التحكم في الوصول▼
Identificationالتعريف
Verifying identity = التحقق من هوية المستخدم (من أنت؟)
Authenticationالمصادقة
Confirming identity = تأكيد وإثبات الهوية (هل أنت فعلاً من تدّعي؟)
Authorizationالتفويض / التصريح
What actions are allowed = تحديد الإجراءات المسموح بها
💡 الآليات (Mechanisms)
• Passwords = كلمات المرور
• MFA = Multi-factor Authentication = المصادقة متعددة العوامل
• Biometrics = القياسات الحيوية (بصمة، عين)
• ACLs = Access Control Lists = قوائم التحكم في الوصول
• RBAC = Role-Based Access Control = التحكم بناءً على الدور
• Least Privilege = الحد الأدنى من الصلاحيات
4Network Security — أمن الشبكة (كاستراتيجية تخفيف)▼
💡 المكونات + أفضل الممارسات
Key Components (المكونات الرئيسية):
• Firewalls = جدران الحماية
• IDS/IPS = Intrusion Detection/Prevention Systems = أنظمة كشف/منع الاختراق
• VPNs = Virtual Private Networks = الشبكات الافتراضية الخاصة
• Network Segmentation = تقسيم الشبكة إلى أجزاء معزولة
• Wireless Security = أمن الشبكة اللاسلكية
• Network Monitoring = مراقبة الشبكة

Best Practices:
• تحديث Firmware (البرامج الثابتة) بانتظام
• استخدام Strong Encryption (تشفير قوي)
• مراقبة Network Traffic (حركة الشبكة)
5Data Security — أمن البيانات (كاستراتيجية تخفيف)▼
💡 الإجراءات + اللوائح
• Data Classification = تصنيف البيانات حسب الحساسية
• Encryption = التشفير: at rest (في التخزين) و in transit (أثناء النقل)
• DLP = Data Loss Prevention = منع فقدان البيانات
• Data Backup and Recovery = النسخ الاحتياطي والاسترداد

7Incident Response — الاستجابة للحوادث — 6 مراحل▼

Preparation — التحضير

الحالة الطبيعية (Steady State) — الاستعداد قبل وقوع أي حادثة.

Identification — التحديد

🏰 Defense in Depth — الدفاع المتعمق
6الطبقات الست للدفاع المتعمق▼
🇬🇧 التعريف
Implementing multiple security controls at different layers to provide redundancy and increase resilience. Benefit: If one layer fails, others are in place to provide protection.
🇸🇦 الترجمة
تطبيق ضوابط أمنية متعددة على طبقات مختلفة لتوفير Redundancy (التكرار/الاحتياطية) وزيادة Resilience (المرونة والصمود).
الفائدة: إذا فشلت طبقة واحدة، توجد طبقات أخرى للحماية.

Physical Security — الأمن المادي

حماية البنية التحتية: أقفال (Locks)، كاميرات مراقبة (CCTV)، حراسة (Guards).

Perimeter Security — أمن المحيط

حماية حدود الشبكة: Firewalls (جدران الحماية)، DMZ (المنطقة منزوعة السلاح).

Network Security — أمن الشبكة

Network Segmentation (تقسيم الشبكة) يمنع انتشار الهجوم Laterally (أفقياً).

Host Security — أمن الجهاز المضيف

Antivirus + Host-based IDS + Patch Management + Encryption (تشفير).

Application Security — أمن التطبيقات

WAF = Web Application Firewall (جدار حماية تطبيقات الويب) + Secure Coding + Input Validation.

Data Security — أمن البيانات (الأكثر داخلية)

Encryption at rest/in transit (تشفير عند التخزين والنقل) + DLP (منع فقدان البيانات).
